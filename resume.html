Entrepreneurial Experience

Canary San Francisco, CA · canaryup.com

Founder January 2013 — Now

Ruby, Rails 4.x, HTML, CSS, JS, Postgres, Sidekiq, Heroku, Git

Canary is a service that notifies you if your website or app goes down. It does this by pinging URLs every minute. If the server fails to respond or responds unsuccessfully, Canary sends email and SMS notifications. The entire product, from design to engineering, has been built by and maintained by myself and my co-founder.

  • CSS Animation: Canary is the first project that I’ve had the chance to use CSS animation in a pratical setting. It contributes significantly to the overall design.

Sqoot Chicago, IL · sqoot.com

Founder January 2011 — December 2012

Ruby, Rails 3.x, HTML, CSS, JS, XSS, Solr, Memcached, Mongo, Redis, Amazon S3, Heroku, Postgres, Git

Sqoot is a local deal API that makes it easy for developers and online publishers to monetize their apps and websites. We integrated with almost 100 local deal providers. For each provider, we would aggregate and normalize deal data, which we’d then expose through our API. The API allows users to filter deals by location, category, and price. The API also allowed customers to track impressions, clicks, and purchases in real-time. Sqoot was acquired by Clutch in 2013.

  • Cross-site Scripting: I built an embeddable widget, which loaded local deals from our servers onto publisher websites. We served millions of deals through these widget installations but ultimately ended up scraping the feature because it couldn’t compete with traditional advertising.
  • Scale: Every day, our API served tens of thousands of requests. In the background, our import system was continuously downloading and updating deals from almost 100 disparate sources.
  • Data Normalization: With almost 100 data sources, the biggest challenge was parsing and normalizing each data set. Background processes would clean, categorize, and localize every deal and merchant as it entered the system.

Managing CSS & JS with Rack March 2011

In most web apps, JS and CSS are a mess. There are few conventions when it comes to organizing these assets and modularizing them can introduce the HTTP overhead of multiple requests. In this talk, I discuss a Rack-based approach to packaging assets using a lightweight gem called Stork. I covered organizational conventions, advanced CSS techniques with Sass, and simple ways to modularize your JS all while sending a tightly packaged set of assets to the browser.
